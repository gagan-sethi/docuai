# Invonix Production Readiness Verification

Date: 2026-07-17

## Official sources of truth

| Metric | Official backend source | Public API |
|---|---|---|
| Revenue, expenses, VAT, P&L, dashboard financial KPIs | `docuai-api/src/lib/financialReporting.ts` | `GET /api/finance/report` |
| OCR statistics | `OcrMonitoring`, written by `src/lib/monitoring.ts` | `/api/monitoring/*` |
| Document/OCR/page/WhatsApp plan usage | Workspace-owner usage reservation in `src/lib/planLimits.ts` | `GET /api/plan` |
| Subscription statistics | `Subscription` and immutable `Payment` records | `/api/admin/subscriptions/stats` |
| Operational document KPIs | `src/lib/analytics.ts` (non-accounting metrics only) | `GET /api/documents/analytics` |

The accounting eligibility rule is: selected workspace/company, `status=approved`,
financial summary present, reconciliation valid, and type is sales invoice,
expense invoice, or receipt. Purchase orders and review/invalid documents are
excluded. Revenue and expenses are VAT-exclusive; VAT is reported separately;
currencies are never combined.

## Resolution by requested item

1. **Single source of truth** — Financial Dashboard, VAT and P&L now call
   `/api/finance/report`. That endpoint runs one query and derives totals,
   monthly rows and categories through `financialReporting.ts`.
2. **Financial reconciliation** — Summary, Report, Monthly and JSON Export use
   the same aggregation functions. The integration suite asserts equality
   between their responses.
3. **Backend vs frontend** — financial aggregation was removed from the three
   financial pages. The browser receives aggregates and only formats them.
4. **Finance Summary API** — the API returns a machine-readable `definition`
   describing eligibility and accounting basis. This documents why Review
   documents do not appear.
5. **OCR validation** — canonical extraction records reconciliation issues;
   approval rejects invalid totals/VAT/line items. Finance queries additionally
   require `financial.reconciliation.isValid=true`.
6. **Duplicate invoices** — processing creates a normalized SHA-256 identity
   from supplier, invoice number, invoice date and gross amount. Matches are
   forced to Review and approval returns `409 DUPLICATE_INVOICE`.
7. **AI expense categorization** — processing invokes the configured AI for
   expense invoices/receipts, persists category confidence, and supplies prior
   approved/manual supplier selections as learning hints. Live multi-supplier
   quality measurement still requires labelled provider fixtures and credentials.
8. **Usage counters** — upload and WhatsApp use the same workspace-owner atomic
   reservation/release path; plan usage reads those same counters. OCR attempts
   are independently recorded in `OcrMonitoring`.
9. **Performance** — financial pages no longer download up to 1,000 documents.
   The report API projects only required fields and performs one bounded server
   query. Exports are generated by the backend from the same eligible rows.
10. **Final verification** — local backend build and 14-test regression suite
    pass. Frontend TypeScript passes. A deployment-data integrity audit, labelled
    OCR/AI evaluation, and browser E2E against staging remain release gates.
